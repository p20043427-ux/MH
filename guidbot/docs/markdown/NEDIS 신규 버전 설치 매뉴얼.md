2. NEDIS 운영
○ NEDIS에 자료를 전송하고 있는 각 응급의료기관은 최소한 아래의 역할 구분에
따라 운영 팀을 구성하여 효율적으로 운영하도록 한다. 담당자 변경 시에는
중앙응급의료센터에 통보하고 데이터 누락 건 등에 대해 담당자가 바로 통보를
받을 수 있도록 하여 정보가 실시간 업데이트 되도록 하여야 한다.
[그림 1] NEDIS 운영을 위한 구성 및 역할
각 응급의료기관에서는 NEDIS 운영팀(관리책임자, 전산운영담당자, 입력담당자)을
구성하고 운영․관리해야 하며, 자체 전송 프로그램 매뉴얼을 구비해야 한다.
기관 내에서 자체적으로 원인 파악이 불가한 문제가 발생한 경우 중앙응급의
료센터로 연락하여 NEDIS Agent PC 재구성을 위한 기술 지원, 기타 운영에
대한 상담 및 지원을 요청할 수 있다. 해당 지침서는 의료기관의 편의를
위하여 시스템 구성설치, 시스템 관리 및 유지와 정보입력의 순서로 분리하여
만들어졌다.

❶ 응급의료진료 전송 설정
n 전송간격은 기존 전송 시간의 설정 값으로 표준전송정보, 외상전송 정
보의 전송간격을 정의할 수 있다.
n 파일연동 설정은 기존버전 모듈의 표준전송의 파일연동 설정과 같은
정보로 파일연동을 통해 표준정보를 전송한 기관을 기존의 설정이 그대로
정의 되어 있다.
❷ 실시간 병상정보 전송 설정
n 전송간격은 실시간 병상정보의 전송 간격 설정 값으로, 최초 설치 시
기본 값인 10초로 정의 되어 있으며, 간격 변경 시 실시간병상정보의 전
송 간격만 변경된다.
n 파일연동 설정은 실시간병상정보의 FTP 전송을 대체하기 위한 기능으
로 기존 FTP로 실시간 병상정보를 전송하던 기관에서 별도의 설정을 통해
Agent를 통해 정보를 전송할 수 있도록 하였다.
n 테스트 전송은 실시간 병상정보의 실제 전송 전 테스트 전송을 통해
전송 대상 정보의 전송 가능여부를 확인할 수 있는 기능이다.
※ 해당 기능은 실시간병상 정보 전송만 제공되는 기능임.
- 해당 메뉴는 신규로 실시간 병상정보를 전송하는 기관이 전송가능여부
를 확인하는데 사용할 수 있는 기능으로 환경설정 화면에서 테스트 전송
을 Y로 설정하게 되면 전송모듈이 테스트 전송모드로 구동되며, 테스트
전송 테이블에 데이터를 입력하면, 테스트 전송을 통해 입력 정보의 전송
여부를 확인 할 수 있다. 테스트 전송 정보는 중앙서버에 기록되지 않는
다.
※ 환경설정 저장 시 전송모듈별 설정정보 변경 사항이 발생하는 경우 해당
전송 모듈을 자동으로 재기동하도록 기능이 개선되었다.
